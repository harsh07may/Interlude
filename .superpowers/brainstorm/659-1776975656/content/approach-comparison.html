Three Approaches for Journal Digitization

How should the capture, processing, and results flow work?

A

Single-Page App with Inline Results

Workflow: Upload photo → OCR processes instantly → Results appear below on same screen

Mobile: Full-screen, scrollable. Take photo → see results immediately

Desktop: Two-column: upload on left, results on right

Pros: Simple, instant feedback

Cons: Small screens may feel cramped during large uploads

B

Multi-Step Modal Workflow

Workflow: Main dashboard → Click "Add Entry" → Modal opens → Upload → Processing → Results in modal

Mobile: Clean dashboard, modal takes full screen for capture

Desktop: Dashboard + centered modal, secondary panels for settings/history

Pros: Organized, scalable to future features

Cons: Extra clicks, feels heavier for simple task

C

Drag-and-Drop with Live Preview

Workflow: Drag image onto canvas → OCR preview updates live → Copy results or save

Mobile: Tap to upload, card-based results that stack

Desktop: Large drop zone on left, results stream on right

Pros: Fast, intuitive, batch-friendly

Cons: Needs good desktop UX, mobile less obvious

Note: MVP only needs capture + OCR extraction. No dashboard/history in Phase 1. Think of this as the core user flow.
